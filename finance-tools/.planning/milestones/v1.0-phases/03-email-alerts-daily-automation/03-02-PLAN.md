**3. Main dispatch** — after the existing `print_summary(forecast)` call (line ~526) and BEFORE `sys.exit(determine_exit_code(forecast))`, add email dispatch logic. The structure should be:

```python
    # Email modes (Phase 3)
    email_requested = args.email_summary or args.test_alert
    if email_requested or args.dry_run:
        if not _has_alert_email:
            print("Error: alert_email.py module not found. Cannot send emails.", file=sys.stderr)
            sys.exit(EXIT_ERROR)

        accounts_config = {a["id"]: a for a in config.get("accounts", [])}

        # Load SMTP config once if we will be sending (not dry-run only)
        smtp_config = None
        recipient = None
        if email_requested and not args.dry_run:
            smtp_config = get_smtp_config()
            recipient = get_alert_recipient()

        if args.email_summary:
            # D-07: Daily summary mode — condensed digest on good days, full report on bad days
            subject, html_body = build_summary_html(forecast)

            if args.dry_run:
                # D-10: Export HTML preview instead of sending
                export_preview(html_body)
            else:
                send_email(subject, html_body, recipient, smtp_config)
                print(f"Summary email sent to {recipient}")

            # Also check for shortfall alerts with dedup (D-01 + D-05)
            alertable = check_alert_thresholds(forecast, accounts_config)
            if alertable and not args.dry_run:
                alert_html = build_alert_html(forecast, alertable)
                alert_subject = f"SHORTFALL ALERT -- {len(alertable)} account(s) need attention"
                accounts_to_alert = []
                for acct in alertable:
                    payment_amounts = [p["amount"] for p in acct["payments"]]
                    alert_hash = compute_alert_hash(acct["id"], acct["projected_balance"], payment_amounts)
                    if should_send_alert(acct["id"], alert_hash):
                        accounts_to_alert.append((acct, alert_hash))

                if accounts_to_alert:
                    send_email(alert_subject, alert_html, recipient, smtp_config)
                    for acct, alert_hash in accounts_to_alert:
                        record_alert_sent(acct["id"], alert_hash)
                    print(f"Shortfall alert sent for {len(accounts_to_alert)} account(s)")
                else:
                    print("Shortfall alerts suppressed (already sent for current conditions)")

        elif args.test_alert:
            # D-09: Force send regardless of shortfalls — test full pipeline
            alertable = check_alert_thresholds(forecast, accounts_config)
            if alertable:
                html_body = build_alert_html(forecast, alertable)
                subject = f"[TEST] SHORTFALL ALERT -- {len(alertable)} account(s)"
            else:
                subject, html_body = build_summary_html(forecast)
                subject = f"[TEST] {subject}"

            if args.dry_run:
                export_preview(html_body)
            else:
                send_email(subject, html_body, recipient, smtp_config)
                print(f"Test alert sent to {recipient}")

        elif args.dry_run:
            # D-10: Dry-run alone — preview summary email
            subject, html_body = build_summary_html(forecast)
            export_preview(html_body)
```

| Boundary | Description |
|----------|-------------|
| CLI args -> payment_forecast.py | User-controlled flags parsed by argparse (validated by argparse itself) |
| payment_forecast.py -> alert_email.py | Internal function calls with forecast data (trusted) |
| .env -> SMTP credentials | Loaded from filesystem, sent to Gmail over TLS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | CLI flag injection | accept | argparse handles flag parsing safely; no shell injection vector. Boolean flags only (store_true) |
| T-03-08 | Information Disclosure | Email content over SMTP | mitigate | STARTTLS encrypts connection to Gmail; credentials never logged; forecast data is personal finance (acceptable risk for self-to-self email) |
| T-03-09 | Denial of Service | --test-alert spam | accept | Single-user tool; user controls invocation frequency. No rate limiting needed |
| T-03-10 | Repudiation | Alert sent but not recorded | mitigate | record_alert_sent() called AFTER successful send_email(); atomic write ensures state persistence |
</threat_model>
