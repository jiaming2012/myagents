**Researched:** 2026-05-04
**Domain:** Xero Accounting API / OAuth2 token management / Python SDK integration
**Confidence:** HIGH

## Summary

This phase replaces Mercury as the business account balance source with Xero. The existing yumyums Xero integration at `~/projects/yumyums/accounting/xero/app.py` provides a proven reference implementation for OAuth2 token persistence and AccountingApi usage. The `xero-python` SDK (v10.0.0) is already installed in the environment.

The primary approach uses `AccountingApi.get_accounts()` to list all accounts filtered by type (BANK and credit card types), then `AccountingApi.get_report_bank_summary()` to retrieve current balances. A key constraint is that the existing Xero OAuth2 token scope does NOT include `accounting.reports.read` -- it only has `accounting.settings.read`, `accounting.banktransactions`, etc. This means either (a) the scope must be expanded by re-authorizing through the yumyums Flask app with the additional scope, or (b) an alternative balance-fetching approach must be used (e.g., computing balance from transactions, or using the Xero statement closing balance).

The codebase follows a clear pattern: standalone `*_balances.py` modules with a single `fetch_*_balances()` export returning `{name: {"balance": float, "type": str}}`. The new `xero_balances.py` mirrors `monarch_balances.py` exactly.

**Primary recommendation:** Use `AccountingApi.get_report_bank_summary()` for balances (requires adding `accounting.reports.read` to OAuth scope). Copy token persistence from yumyums verbatim. Mirror `monarch_balances.py` structure.

### Pattern 1: Token Persistence (copy from yumyums)
**What:** File-based OAuth2 token storage with automatic refresh
**When to use:** Every Xero API call
**Example:**
```python
# Source: ~/projects/yumyums/accounting/xero/app.py (verified)
TOKEN_FILE = Path(__file__).parent / ".xero_token.json"
TOKEN_URL = "https://identity.xero.com/connect/token"

def load_token():
    if TOKEN_FILE.exists():
        return json.loads(TOKEN_FILE.read_text())
    return None

def save_token(token):
    TOKEN_FILE.write_text(json.dumps(token))

def refresh_token(token):
    response = requests.post(
        TOKEN_URL,
        data={
            "grant_type": "refresh_token",
            "refresh_token": token["refresh_token"],
        },
        auth=(CLIENT_ID, CLIENT_SECRET),
        headers={"Content-Type": "application/x-www-form-urlencoded"},
    )
    response.raise_for_status()
    new_token = response.json()
    save_token(new_token)
    return new_token
```

### Pattern 2: AccountingApi Usage (from yumyums)
**What:** Initialize SDK client, get tenant ID, call API
**When to use:** Fetching accounts/balances

### Anti-Patterns to Avoid
- **Using `auto-setup` token from yumyums directly:** The token file contains sensitive credentials. Copy the PATTERN, not the file. The banking CLI gets its own `.xero_token.json` via the yumyums Flask app's initial auth.
- **Calling Xero API without refresh logic:** Access tokens expire in 30 minutes. Always wrap API calls in try/except with automatic token refresh on 401.
- **Relying on `get_accounts()` for balances:** The Account model does NOT have a balance field. You must use `get_report_bank_summary()` or compute from transactions.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| OAuth2 token refresh | Custom OAuth2 flow | Copy yumyums `refresh_token()` pattern with raw `requests` | Proven pattern, handles edge cases (expired refresh token -> redirect to login) |
| Xero API client setup | Raw HTTP calls to Xero endpoints | `xero-python` SDK's `ApiClient` + `AccountingApi` | SDK handles serialization, auth headers, pagination, error models |
| Tenant ID resolution | Hardcoded tenant ID | `IdentityApi.get_connections()[0].tenant_id` | Tenant ID changes if org is disconnected/reconnected |

**Key insight:** The yumyums Xero integration is a complete reference implementation. Copy patterns wholesale rather than reinventing.

## Common Pitfalls

### Pitfall 1: OAuth2 Scope Mismatch
**What goes wrong:** API calls return 403 Forbidden because the token lacks required scopes
**Why it happens:** The existing yumyums token has `accounting.settings.read` but NOT `accounting.reports.read` (needed for `get_report_bank_summary`)
**How to avoid:** Re-authorize through yumyums Flask app with expanded SCOPE string that includes `accounting.reports.read`. Alternatively, use `get_accounts()` to list accounts and compute balance from statement data or use a different endpoint.
**Warning signs:** 403 status code from Xero API on report endpoints
[VERIFIED: Token scope checked from .xero_token.json]

### Pitfall 2: Xero Balances Are Accounting Balances, Not Bank Balances
**What goes wrong:** Xero shows a different balance than what the bank website shows
**Why it happens:** Xero balances reflect reconciled transactions in Xero, not the live bank feed balance. Unreconciled transactions won't be included.
**How to avoid:** Accept this limitation -- Xero accounting balance is close enough for forecast purposes. The bank summary report closing balance is the most current figure available via API.
**Warning signs:** Balance discrepancy between Xero and direct bank login
[ASSUMED]

### Pitfall 3: Token File Shared Between Projects
**What goes wrong:** yumyums and banking CLI both try to use/refresh the same token file, causing race conditions
**Why it happens:** If `.xero_token.json` path points to the same file
**How to avoid:** Per D-01, the banking CLI gets its OWN `.xero_token.json` in the banking repo root. Both share `XERO_CLIENT_ID`/`SECRET` env vars but maintain separate token files.
**Warning signs:** Intermittent 401 errors in one project after the other refreshes

### Pitfall 4: Refresh Token Expiration
**What goes wrong:** Xero refresh tokens expire after 60 days of non-use
**Why it happens:** If the CLI isn't run for 60+ days, the refresh token becomes invalid
**How to avoid:** Document that re-authorization via yumyums Flask app is needed if refresh fails. The `refresh_token()` function should raise a clear error pointing the user to re-auth.
**Warning signs:** `refresh_token()` returns 400 with "invalid_grant"
[ASSUMED]

### Pitfall 5: Business Accounts Not All in Xero
**What goes wrong:** Only Mercury-banked business accounts may be visible in Xero. Business accounts at BoA, Chase, Lafayette may not appear.
**Why it happens:** Xero only knows about accounts connected to it via bank feeds or manual setup. The 9 business accounts in payments.yaml span multiple banks.
**How to avoid:** The `xero_account_id` field in payments.yaml is only populated for accounts that exist in Xero. `resolve_balance()` falls back to Monarch for business accounts without `xero_account_id`.
**Warning signs:** Fewer accounts returned from Xero than expected
[ASSUMED]

### Pitfall 6: Credit Card Balances Are Negative in Xero
**What goes wrong:** Credit card balances from Xero may be negative (representing what you owe)
**Why it happens:** Xero uses accounting convention where credit card liability is negative
**How to avoid:** Check the account type in the response and use `abs()` for credit cards if the forecast needs positive "amount owed" figures. This mirrors Pitfall 6 in the existing codebase (monarch_balances).
**Warning signs:** Negative balance values for credit card accounts
[ASSUMED]

```python
# Source: Synthesis of yumyums/accounting/xero/app.py + monarch_balances.py patterns
#!/usr/bin/env python3
"""Fetch business account balances from Xero Accounting API."""

import json
import os
import sys
from pathlib import Path

try:
    import requests
except ImportError:
    print("Error: 'requests' package required.", file=sys.stderr)
    sys.exit(1)

try:
    from xero_python.api_client import ApiClient, Configuration
    from xero_python.api_client.oauth2 import OAuth2Token
    from xero_python.accounting import AccountingApi
    from xero_python.identity import IdentityApi
except ImportError:
    AccountingApi = None

CLIENT_ID = os.getenv("XERO_CLIENT_ID")
CLIENT_SECRET = os.getenv("XERO_CLIENT_SECRET")
TOKEN_URL = "https://identity.xero.com/connect/token"
TOKEN_FILE = Path(__file__).parent / ".xero_token.json"

# --- Token persistence (copied from yumyums pattern) ---

def load_token():
    if TOKEN_FILE.exists():
        return json.loads(TOKEN_FILE.read_text())
    return None

def save_token(token):
    TOKEN_FILE.write_text(json.dumps(token))

def refresh_token(token):
    response = requests.post(
        TOKEN_URL,
        data={"grant_type": "refresh_token", "refresh_token": token["refresh_token"]},
        auth=(CLIENT_ID, CLIENT_SECRET),
        headers={"Content-Type": "application/x-www-form-urlencoded"},
    )
    response.raise_for_status()
    new_token = response.json()
    save_token(new_token)
    return new_token

# --- Balance fetching ---

def fetch_xero_balances():
    """Fetch bank + credit card balances from Xero.

    Returns: {account_name: {"balance": float, "type": str}}
    """
    if AccountingApi is None:
        print("Warning: xero-python not installed, skipping Xero balance lookup", file=sys.stderr)
        return {}

    if not CLIENT_ID or not CLIENT_SECRET:
        print("Warning: XERO_CLIENT_ID/SECRET not set, skipping Xero", file=sys.stderr)
        return {}

    token = load_token()
    if not token:
        print("Warning: No Xero token found. Run yumyums Flask app to authorize.", file=sys.stderr)
        return {}

    # Refresh token proactively
    try:
        token = refresh_token(token)
    except Exception as e:
        print(f"Warning: Xero token refresh failed: {e}", file=sys.stderr)
        return {}

    # Initialize SDK
    api_client = ApiClient(
        Configuration(oauth2_token=OAuth2Token(client_id=CLIENT_ID, client_secret=CLIENT_SECRET))
    )
    api_client.set_oauth2_token(token)

    # Get tenant ID
    identity_api = IdentityApi(api_client)
    connections = identity_api.get_connections()
    if not connections:
        print("Warning: No Xero tenants connected.", file=sys.stderr)
        return {}
    tenant_id = connections[0].tenant_id

    # Get accounts (BANK type)
    accounting_api = AccountingApi(api_client)
    result = accounting_api.get_accounts(xero_tenant_id=tenant_id)

    # Get bank summary for balances
    report = accounting_api.get_report_bank_summary(xero_tenant_id=tenant_id)

    # Parse report rows into balances dict
    # (Report structure needs runtime inspection -- see Open Questions)
    balances = {}
    # ... parse report.rows to extract account names and closing balances
    return balances
```

### resolve_balance() Update Pattern
```python
# Source: coverage_report.py (modified)
def resolve_balance(account, monarch_balances, xero_balances):
    """Get the current balance for an account from available sources."""
    # Try Xero first (for business accounts with xero_account_id)
    xero_id = account.get("xero_account_id")
    if xero_id and xero_id in xero_balances:
        return xero_balances[xero_id]["balance"], "xero"

    # Try Monarch
    monarch_match = account.get("monarch_match")
    if monarch_match and monarch_match in monarch_balances:
        return monarch_balances[monarch_match]["balance"], "monarch"

    # Partial match on Monarch (displayName contains last4)
    last4 = account.get("last4")
    if last4:
        for name, data in monarch_balances.items():
            if last4 in name:
                return data["balance"], "monarch"

    return None, None
```

## Open Questions

1. **Bank Summary Report Response Structure**
   - What we know: `get_report_bank_summary()` returns `ReportWithRows` object. [VERIFIED: SDK method signature]
   - What's unclear: The exact structure of `rows` -- which row/cell indices contain account names vs closing balances. This is a tabular report, not a typed model.
   - Recommendation: Implement with runtime inspection. Call the API once, print the raw response, map fields accordingly. This is a 5-minute task during implementation.

2. **OAuth2 Scope Expansion**
   - What we know: Current token scope is `openid profile email accounting.invoices accounting.payments accounting.banktransactions accounting.manualjournals accounting.settings.read accounting.contacts offline_access`. [VERIFIED: .xero_token.json]
   - What's unclear: Whether `accounting.reports.read` is sufficient post-March 2026, or if Xero requires new granular scopes. [ASSUMED based on web search -- Xero changed scope policy March 2, 2026]
   - Recommendation: Add `accounting.reports.read` to the yumyums Flask app SCOPE string and re-authorize. If 403 persists, check Xero developer console for required granular scopes.

3. **Which Business Accounts Exist in Xero**
   - What we know: 9 business accounts in payments.yaml across Mercury, BoA, Chase, Lafayette Federal. The Xero org is "Yumyums Kitchen LLC".
   - What's unclear: Which of these banks have bank feeds connected in Xero. Mercury accounts almost certainly do. BoA/Chase/Lafayette may or may not.
   - Recommendation: Run `fetch_xero_balances()` once after implementation to discover available accounts, then populate `xero_account_id` fields in payments.yaml for matching accounts only.

4. **Cache Strategy for Xero Balances**
   - What we know: Existing codebase uses file-based caching in `.cache/` with TTLs (token: 50min, events: 15min, details: 24hr). [VERIFIED: zoho_calendar_payments.py]
   - What's unclear: Optimal cache TTL for balance data.
   - Recommendation: Use same cache infrastructure. Set balance cache TTL to 15 minutes (same as event list) -- balances don't change frequently for a forecast tool.
