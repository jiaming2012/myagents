# Phase 01.1: Xero Business Balance Integration - Context

**Gathered:** 2026-05-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace Mercury with Xero as the business account balance source. Create a new `xero_balances.py` module that fetches bank and credit card balances from Xero via the `xero-python` SDK, using the same OAuth2 token persistence pattern from the existing yumyums Xero integration. Update `coverage_report.py` to use Xero instead of Mercury for business accounts. Update `payments.yaml` to replace `mercury_id` with `xero_account_id` for business accounts.

</domain>

<decisions>
## Implementation Decisions

### Token Management
- **D-01:** Copy the token persistence pattern from `~/projects/yumyums/accounting/xero/app.py` — `load_token()`, `save_token()`, `refresh_token()` functions with `.xero_token.json` file-based storage.
- **D-02:** CLI manages its own token file. Initial OAuth2 authorization done via the yumyums Flask app or a one-time setup. The CLI only needs refresh capability.
- **D-03:** Xero env vars: `XERO_CLIENT_ID`, `XERO_CLIENT_SECRET` (same as yumyums app). No `XERO_BANK_ACCOUNT_ID` needed — we fetch all accounts.

### Account Matching
- **D-04:** Add `xero_account_id` field to business accounts in `payments.yaml`, replacing `mercury_id`. Name-based matching similar to Monarch's `monarch_match` pattern.
- **D-05:** `resolve_balance()` in `coverage_report.py` updated: try Xero first for accounts with `xero_account_id`, then fall back to Monarch. Mercury code removed.
- **D-06:** Match Xero accounts to payments.yaml by account name/nickname (same strategy as Monarch matching via `monarch_match` field).

### Balance Fetching Scope
- **D-07:** Fetch both bank account AND credit card balances from Xero (AccountType.BANK and credit card types).
- **D-08:** Return format matches existing pattern: `{account_name: {"balance": float, "type": str}}` — consistent with `fetch_monarch_balances()` return shape.

### Code Organization
- **D-09:** New `xero_balances.py` module in repo root. Exports `fetch_xero_balances()`. Follows the `monarch_balances.py` pattern (standalone module, single responsibility).
- **D-10:** `coverage_report.py` imports from `xero_balances` instead of Mercury functions. `zoho_calendar_payments.py` import updated similarly.
- **D-11:** `xero-python` SDK added to requirements. Token persistence uses `requests` (already a dependency) for raw HTTP refresh calls — same as the yumyums pattern.

### Claude's Discretion
- Whether to use the `xero-python` SDK's `AccountingApi` directly or raw HTTP calls for balance fetching (SDK preferred if straightforward)
- Cache TTL for Xero balance data (follow existing patterns in the codebase)

</decisions>
