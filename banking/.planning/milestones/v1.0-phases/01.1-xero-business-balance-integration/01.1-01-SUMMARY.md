# Phase 01.1 Plan 01: Xero Balance Module + Config Summary

Xero balance fetching module with OAuth2 token persistence (copied from yumyums pattern) and config updates replacing Mercury with Xero account identifiers.

## What Was Built

### Task 1: xero_balances.py module
Created standalone `xero_balances.py` in repo root exporting `fetch_xero_balances()`. The module follows the `monarch_balances.py` pattern with:
- Token persistence (`load_token`, `save_token`, `refresh_token`) copied verbatim from yumyums Xero integration
- Proactive token refresh before every API call (30-min expiry)
- SDK initialization: `ApiClient` -> `IdentityApi.get_connections()` for tenant ID -> `AccountingApi.get_report_bank_summary()` for balances
- `_parse_bank_summary_report()` helper that extracts account names and closing balances from `ReportWithRows` response
- Graceful degradation: returns `{}` with stderr warnings on any failure (missing SDK, missing creds, missing token, API errors)
- 401 retry logic: attempts one token refresh on HTTP 401 before giving up
- Clear error message for expired refresh tokens pointing user to re-auth via yumyums Flask app
- Standalone `main()` function for manual testing via `python xero_balances.py`

### Task 2: Config file updates
- **payments.yaml**: Replaced `mercury_id`/`mercury_key` with `xero_account_id: null` on all 9 business accounts. Removed `mercury_id`/`mercury_key` from 2 personal Mercury accounts. Updated header comment.
- **requirements.txt**: Added `xero-python` dependency
- **.gitignore**: Added `.xero_token.json` (OAuth2 token security -- T-01.1-01 mitigation)
