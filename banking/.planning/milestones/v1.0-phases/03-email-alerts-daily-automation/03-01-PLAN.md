**Function 1: get_smtp_config()** — per D-03, load from .env:
- Read SMTP_HOST (default "smtp.gmail.com"), SMTP_PORT (default "587"), SMTP_USER (required), SMTP_PASSWORD (required)
- Raise RuntimeError listing missing vars if SMTP_USER or SMTP_PASSWORD is None
- Return dict with keys: host, port (int), user, password

**Function 2: get_alert_recipient()** — per D-03:
- Read ALERT_EMAIL from env
- Raise RuntimeError if not set
- Return string

**Function 3: check_alert_thresholds(forecast, accounts_config)** — per D-01:
- `forecast`: dict from build_forecast()
- `accounts_config`: dict mapping account_id -> account dict from payments.yaml
- For each account in forecast["accounts"]:
  - Look up alert_on from accounts_config, default to "error" if missing (per D-01)
  - If alert_on == "none": skip
  - If alert_on == "error" and severity == "error": include
  - If alert_on == "warning" and severity in ("error", "warning"): include
- Return list of forecast account dicts that meet their threshold

**Function 4: build_alert_html(forecast, alertable_accounts)** — per D-02, D-04:
- Build full forecast HTML table with ALL accounts (per D-04: "full forecast with problem accounts highlighted")
- Highlight accounts that appear in alertable_accounts:
  - severity == "error": background #fee2e2 (light red), bold text
  - severity == "warning": background #fef9c3 (light yellow)
  - severity == "ok": background #ffffff (white)
- Table columns: Account, Current Balance, Outgoing, Projected Balance
- Include summary section at bottom: Total Outgoing, Total Available, Net Position
- All CSS inline on elements (Gmail strips style tags per research)
- Use table-based layout, font-family: Arial, Helvetica, sans-serif
- Header row background: #f3f4f6
- Cell padding: 8px, border: 1px solid #ddd
- Wrap in <html><body> tags
- Return HTML string

**Function 5: build_summary_html(forecast)** — per D-07:
- Check if any account has severity != "ok"
- If all ok (good day): build CONDENSED digest — just summary totals table (Total Outgoing, Total Available, Net Position) with a "All accounts healthy" header
- If any warning/error (problem day): build FULL report — same as build_alert_html but called with all accounts as alertable, plus summary section. Use build_alert_html(forecast, [a for a in forecast["accounts"] if a["severity"] != "ok"]) internally
- Subject line prefix: "Forecast Summary" for good day, "Forecast ALERT" for problem day
- Return tuple: (subject, html_body)

**Function 6: compute_alert_hash(account_id, projected_balance, payment_amounts)** — per D-05:
- Build JSON string from dict: {"account_id": account_id, "projected_balance": round(projected_balance, 2), "payments": sorted(round(a, 2) for a in payment_amounts)}
- Use sort_keys=True for determinism
- Return hashlib.sha256(data.encode()).hexdigest()[:16]

**Function 7: should_send_alert(account_id, alert_hash)** — per D-05:
- Load .alert_state.json (return True if file doesn't exist)
- Compare alert_hash to state[account_id]["hash"]
- Return True if different or account not in state

**Function 8: record_alert_sent(account_id, alert_hash)** — per D-06:
- Load existing state (empty dict if file missing)
- Set state[account_id] = {"hash": alert_hash, "sent_at": datetime.now().isoformat()}
- Write atomically: write to .alert_state.json.tmp, then os.replace() to .alert_state.json (per Pitfall 3)

**Function 9: _load_state() / _save_state(state)** — internal helpers:
- _load_state(): Read .alert_state.json, return dict. Return {} if file missing or JSON invalid.
- _save_state(state): Write to temp file, os.replace to target. Use json.dumps with indent=2.

**Function 10: send_email(subject, html_body, recipient, smtp_config)** — per research Pattern 1:
- Create MIMEMultipart("alternative")
- Set Subject, From (smtp_config["user"]), To (recipient)
- Attach plain text fallback: "View this email in an HTML-capable client."
- Attach HTML body
- Connect with smtplib.SMTP(host, port, timeout=30) — per Pitfall 4
- Call starttls(), login(user, password), send_message(msg)
- Use context manager (with statement) for auto-cleanup
- Catch smtplib.SMTPException and socket.timeout, print error to stderr, re-raise

**Function 11: export_preview(html_body, filename="forecast_preview.html")** — per D-10:
- Write html_body to Path(filename)
- Print "Preview written to {absolute_path}" to stdout
- Print "Open in browser: file://{absolute_path}" to stdout

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env -> alert_email.py | SMTP credentials loaded from filesystem into memory |
| alert_email.py -> smtp.gmail.com | Credentials sent over TLS to Google SMTP server |
| .alert_state.json -> alert_email.py | Dedup state loaded from filesystem (untrusted if manually edited) |
| build_forecast() -> alert_email.py | Forecast data from Phase 2 engine (trusted internal data) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | .env SMTP_PASSWORD | mitigate | .env is gitignored (existing pattern); never log/print password; use env vars not hardcoded values |
| T-03-02 | Spoofing | send_email From header | accept | Single-user tool; From = SMTP_USER (the authenticated sender). No spoofing risk for self-to-self emails |
| T-03-03 | Tampering | .alert_state.json | accept | Local file, single-user machine. Worst case: tampered state causes duplicate alert (not harmful). Atomic write prevents corruption from concurrent runs |
| T-03-04 | Denial of Service | SMTP timeout | mitigate | Set timeout=30 on smtplib.SMTP(); catch SMTPException and socket.timeout; exit non-zero so cron reports failure |
| T-03-05 | Information Disclosure | forecast_preview.html | accept | Local preview file, gitignored. Contains same data user already sees in CLI output. No new exposure |
| T-03-06 | Elevation of Privilege | payments.yaml alert_on | accept | Config field only affects alert threshold filtering, not access control. Invalid values treated as "error" (default) |
</threat_model>
