    ```python
    def display_grouped_payments(payments, errors, balances=None):
        """Display payments grouped by funding account. Per D-16."""
        print(f"\n{'=' * 60}")
        print(f"  BILL MAP -- {datetime.now().strftime('%a %b %d, %Y')}")
        print(f"{'=' * 60}")

        grouped = {}
        no_funding = []
        for p in payments:
            if p.get("no_funding"):
                no_funding.append(p)
            else:
                key = p.get("fund_account_id", "UNMATCHED")
                grouped.setdefault(key, []).append(p)

        for acct_id in sorted(grouped.keys()):
            acct_payments = grouped[acct_id]
            print(f"\n  {acct_id}")
            print(f"  {'─' * 50}")
            for p in sorted(acct_payments, key=lambda x: x.get("due_date") or ""):
                flag = " ~estimate" if p.get("is_variable") else ""
                print(f"    {p['due_date']}  {p['name']}: ${p['amount']:,.2f}{flag}")

        if no_funding:
            print(f"\n  NO FUNDING ACCOUNT (informational)")
            print(f"  {'─' * 50}")
            for p in no_funding:
                print(f"    {p['due_date']}  {p['name']}: ${p['amount']:,.2f}")

        total = sum(p["amount"] for p in payments)
        print(f"\n  {'─' * 50}")
        print(f"  Total: ${total:,.2f} across {len(payments)} payment(s)")

        if errors:
            print(f"\n  ERRORS ({len(errors)}):", file=sys.stderr)
            for err in errors:
                print(f"    - {err}", file=sys.stderr)
    ```

    **4. Add `resolve_variable_amounts()` function** (per D-11):
    ```python
    async def resolve_variable_amounts(payments, config):
        """For variable payments, replace title amount with real Monarch balance. Per D-11."""
        # Only fetch if there are variable payments
        variable = [p for p in payments if p.get("is_variable") and p.get("source_account")]
        if not variable:
            return payments

        monarch_balances = await fetch_monarch_balances()

        # Build lookup from source_account nickname to Monarch display name
        nickname_lookup = build_nickname_lookup(config)

        for p in variable:
            source = p["source_account"]
            # Find the account in config to get monarch_match
            try:
                acct_id = resolve_account(source, nickname_lookup)
            except ValueError:
                continue  # Can't resolve source account, keep estimate

            # Find account config
            acct_config = next((a for a in config.get("accounts", []) if a["id"] == acct_id), None)
            if not acct_config:
                continue

            # Try to get balance from Monarch
            balance, _ = resolve_balance(acct_config, monarch_balances, {})
            if balance is not None:
                p["amount"] = abs(balance)  # Pitfall 6: credit cards are negative
                p["amount_source"] = "monarch"

        return payments
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Zoho Calendar API -> application | Event data (titles, descriptions) from external API |
| Monarch Money API -> application | Account balance data from external API |
| Mercury API -> application | Account balance data from external API |
| Application -> Zoho Calendar API | PUT request to update event titles (--update-calendar only) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Information Disclosure | env vars in error messages | mitigate | Error messages reference env var names (ZOHO_CLIENT_ID) but never print their values; existing get_required_env() already follows this pattern |
| T-01-06 | Tampering | update_event_title PUT | mitigate | --update-calendar is opt-in (D-12), not default. Requires explicit flag. etag provides optimistic concurrency. Only title field is modified. |
| T-01-07 | Information Disclosure | cached API responses in .cache/ | accept | .cache/ is in .gitignore. Single-user local tool. Cache files contain event metadata, not credentials. TTL expiry limits exposure window. |
| T-01-08 | Spoofing | Monarch/Mercury balance data | accept | API credentials validated at connection time. Balance data used for display only, not financial transactions. Tool is read-only (no money movement). |
| T-01-09 | Elevation of Privilege | payments.yaml nickname injection | accept | Local file edited only by the user. Nicknames are lookup keys, not executable code. safe_load prevents YAML deserialization attacks. |
</threat_model>
