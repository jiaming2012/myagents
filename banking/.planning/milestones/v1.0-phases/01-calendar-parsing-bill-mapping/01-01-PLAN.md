<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Zoho Calendar API -> parse functions | Untrusted event title and description text from external API |
| payments.yaml -> nickname lookup | Trusted local config (user-controlled) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | parse_event_title | mitigate | Strict regex validation (TITLE_PATTERN) rejects any title not matching "Name - $Amount" format; non-matching titles become collected errors, never silently accepted (D-01) |
| T-01-02 | Tampering | parse_event_notes | mitigate | Structured parsing with explicit field prefixes ("Fund:", "Source:") and whitelist keyword ("VARIABLE"); missing Fund: field raises ValueError (D-07) |
| T-01-03 | Information Disclosure | payments.yaml | accept | Local file, single-user tool, no network exposure. Account IDs and nicknames are not sensitive (no account numbers, just display names and last4) |
| T-01-04 | Denial of Service | process_events | accept | CLI tool run manually; no external trigger. Malformed events cause collected errors, not crashes |
</threat_model>
