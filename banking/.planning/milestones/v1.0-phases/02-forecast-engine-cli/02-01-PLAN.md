       - `account_lookup` is dict {account_id: account_dict} built from config["accounts"]
       - If the payment maps to a credit card account (find the account whose payments.yaml `name` or `id` matches -- look for accounts with `type: "credit"` that the payment is paying), use the credit card's Monarch balance via resolve_balance(), take abs() of the balance (Pitfall 1: credit cards are negative in Monarch).

    6. **build_forecast(config, monarch_balances, xero_balances, days)** -- per D-07, D-10, D-11.
       Returns dict with structure:
       ```python
       {
           "accounts": [
               {
                   "id": str,
                   "name": str,
                   "current_balance": float,
                   "balance_source": str,  # "monarch", "xero", or "unknown"
                   "payments": [{"name": str, "amount": float, "due_date": datetime}],
                   "projected_balance": float,  # current_balance - sum(payment amounts)
                   "min_balance": float,  # from account config, default 0
                   "severity": "ok" | "warning" | "error",
               }
           ],
           "summary": {
               "total_outgoing": float,
               "total_available": float,
               "net_position": float,  # total_available - total_outgoing
           }
       }
       ```
       Logic:
       - Build accounts_by_id lookup from config["accounts"]
       - Group payments by funding_account (D-05 guarantees all have one after validation)
       - For each funding account: resolve current balance via resolve_balance(), compute upcoming payment dates via get_payment_dates_in_horizon(), resolve each payment amount via resolve_payment_amount(), sort by due_date
       - projected_balance = current_balance - sum(payment amounts)
       - Severity per D-10: projected < 0 = "error", projected < min_balance (account.get("min_balance", 0)) = "warning", else "ok"
       - Summary per D-09: total_outgoing = sum of all payment amounts across all accounts, total_available = sum of current_balance for accounts with positive balance, net_position = total_available - total_outgoing

    7. **Graceful degradation for balance fetch failure** (discretion area): If resolve_balance returns (None, None) for an account, set balance to 0.0, source to "unknown", and severity to "error" (can't verify solvency without balance). Print a stderr warning.
